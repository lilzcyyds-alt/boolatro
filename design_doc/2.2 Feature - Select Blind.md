# 2.2 Feature - Select Blind

### Blind Configuration
Blinds are now defined in `assets/config/blinds.json`. Each blind entry includes:
- `id`: Unique identifier.
- `name`: Display name.
- `category`: `small blind`, `big blind`, or `boss blind`.
- `targetScore`: Required score to pass.
- `reward`: Money granted on success.
- `premise`: The initial logic sentence.

The `BlindSystem` handles loading these configurations and provides them to the `SelectBlindStageComponent`.

## 目标
玩家的目标是完成 **8 个 Ante**。每个 Ante 包含三个顺序阶段：
1. **Small Blind** (可跳过)
2. **Big Blind** (可跳过)
3. **Boss Blind** (必须击败)

击败 Boss Blind 后，Ante 加 1。完成 Ante 8 后获得最终胜利。
> [!NOTE]
> 当前的 **Ante** 数值不再显示在选择界面标题中，而是统一显示在**计分面板 (Region B)** 的最底部。


## 交互流程与界面表现

### 1. 同屏展示与动态聚焦
在选择阶段，系统会一次性加载所有 3 个盲注卡片（Small, Big, Boss）并水平排列。
- **当前激活项**：Y 轴位置较高，不透明度 1.0，显示交互按钮。
- **已跳过项/待选项**：Y 轴位置较低，不透明度降低（0.5-0.7），按钮隐藏。

### 2. 卡片按钮布局
每个盲注卡片下方垂直排列两个交互按钮：
- **SELECT** (白色)：选择并进入 Proof 阶段。
- **SKIP BLIND** (橙红色)：跳过当前阶段，直接激活下一个盲注。卡片会通过平滑的 `flyTo` (位置) 和 `fadeTo` (不透明度) 动画进行过渡：
    - 当前卡片下移并淡出 (0.5 opacity)。
    - 下一个卡片上移并亮起 (1.0 opacity)。
- **Boss 限制**：Boss 盲注卡片仅显示 SELECT 按钮，不可跳过。

### 3. 难度缩放
每个阶段的目标分数会随 Ante 增加而自动缩放：
`Target Score = Base Score * 1.5 ^ (Ante - 1)`

## 数据结构细节
盲注的定义位于 `assets/config/blinds.json`，增加了 `category` 字段用于逻辑区分：
- `small blind`
- `big blind`
- `boss blind`
